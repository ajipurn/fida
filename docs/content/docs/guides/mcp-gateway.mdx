title: MCP Gateway
description: Serve Fida's redacting MCP tools and gate third-party MCP servers.

Fida exposes gateway tools over MCP:

- `fida_read`: read files through PathJail, policy, redaction, and audit.
- `fida_shell`: run shell commands through policy, optional sandboxing, redaction, and audit.

The built-in default is redaction-first. A sensitive filename does not by itself block `fida_read`; Fida reads the file, replaces detected values, and returns the useful remainder. Model-bound redaction is an invariant: policy can make a read stricter by blocking it, but it cannot make raw detected secret values safe to return. Explicit policy denies and the `strict-firewall` preset can still block a read before disk access.

Start the gateway:

```bash
fida mcp serve
```

Use a specific workspace root:

```bash
fida mcp serve --workspace /path/to/repo
```

## Gateway tools

How `fida_read` and `fida_shell` evaluate each request before anything runs.

### Read flow

```text
MCP fida_read -> PathJail -> file.read policy -> read file -> redact -> audit -> response
```

### Shell flow

```text
MCP fida_shell -> command.run policy -> optional OS sandbox -> execute -> redact -> audit -> response
```

Both flows fail closed on redaction failure: raw content is suppressed instead of returned.

## Third-party servers

Inspect and gate MCP servers you do not control.

### Inspect MCP servers

Fida can inspect and reason about third-party MCP server tools:

```bash
fida mcp inspect server.json
fida mcp list-tools server.json
fida mcp explain-tool browser.navigate
```

### Proxy MCP servers

Use the policy-gated stdio proxy to gate `tools/call` requests before they reach a server:

```bash
fida mcp proxy --server server.json
```

The proxy evaluates `mcp.tool_call` actions and blocks denied or unresolved ask decisions in non-interactive mode.

## Controls

Environment switches that adjust isolation for the gateway.

### PathJail controls

PathJail is enabled by default for gateway reads and shell working directories. It blocks absolute paths and symlink escapes outside the workspace.

Disable only when the surrounding environment already provides isolation:

```bash
FIDA_NO_JAIL=1 fida mcp serve
```

### Sandbox controls

`fida_shell` commands can be wrapped in an OS sandbox for defense in depth, so even a policy-allowed command is network-isolated and blocked from reading credential stores:

```bash
FIDA_SANDBOX=1 fida mcp serve
```

Backends: Seatbelt (`sandbox-exec`) on macOS denies outbound network and reads of `~/.ssh`, `~/.aws`, `~/.gnupg`, and similar stores; bubblewrap (`bwrap`) on Linux isolates the network. It only affects `fida_shell` (reads already pass through PathJail + redaction), is off by default, and is a no-op where no backend exists. Policy still decides whether the command runs at all. Run `fida doctor` to confirm a backend is available; see the [security model](/docs/concepts/security-model) for the full per-platform matrix.
